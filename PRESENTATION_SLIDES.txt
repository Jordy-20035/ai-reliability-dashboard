СЛАЙД 3: Описание проблемы - Обзор
--------------------------------------------------------------------------------
Современные AI-модели сталкиваются с критическими вызовами после развертывания:

• Деградация производительности модели
  - Точность снижается со временем
  - Метрики отклоняются от показателей обучения
  - Модели в продакшене становятся ненадежными

• Дрифт данных
  - Изменение распределения входных данных
  - Смещение распределения признаков
  - Модель расходится с реальностью

• Проблемы справедливости и предвзятости
  - Неравномерные предсказания для разных демографических групп
  - Юридические и этические проблемы
  - Дискриминация в автоматизированных решениях


СЛАЙД 4: Описание проблемы - Реальное влияние
--------------------------------------------------------------------------------
Почему эта проблема важна:

Финансовое влияние:
- Деградированные модели → плохие бизнес-решения
- Предвзятые модели → юридическая ответственность
- Сбои системы → потеря доходов

Этическое влияние:
- Несправедливые исходы для уязвимых групп
- Дискриминация в найме, кредитовании, правосудии
- Потеря общественного доверия к AI-системам

Техническое влияние:
- Необходимость постоянного ручного мониторинга
- Сложно обнаружить проблемы на раннем этапе
- Ограниченные инструменты для комплексного мониторинга

СЛАЙД 6: Существующие решения - Обзор
--------------------------------------------------------------------------------
Текущее состояние инструментов мониторинга MLOps:

Категория 1: Общие платформы MLOps
• MLflow - отслеживание экспериментов, реестр моделей
• Weights & Biases - отслеживание экспериментов
• Kubeflow - ML-процессы на базе Kubernetes

Категория 2: Инструменты обнаружения дрифта
• Evidently AI - открытый исходный код для обнаружения дрифта
• Alibi Detect - обнаружение атак и дрифта
• NannyML - наука о данных после развертывания

Категория 3: Инструменты справедливости
• Fairlearn - оценка и смягчение справедливости
• AIF360 - набор инструментов справедливости от IBM
• What-If Tool - анализ справедливости от Google


СЛАЙД 7: Существующие решения - Ограничения
--------------------------------------------------------------------------------
Пробелы в существующих решениях:

Ограничения:
• Фрагментированные инструменты
  - Разные инструменты для дрифта, справедливости, производительности
  - Нет единой платформы
  - Сложная интеграция

• Требования к ресурсам
  - Многие инструменты требуют GPU или тяжелую инфраструктуру
  - Недоступны для CPU-only окружений
  - Высокие вычислительные затраты

• Ограниченное оповещение в реальном времени
  - Большинство инструментов фокусируются на пакетном анализе
  - Ограниченные возможности мониторинга в реальном времени
  - Ручная настройка оповещений

• Сложность
  - Крутая кривая обучения
  - Сложная установка и настройка
  - Требуется поддержка DevOps

Метрики справедливости:
• Демографический паритет
• Равные возможности
• Коэффициент несопоставимого воздействия

• Известные проблемы: Хорошо документированные проблемы предвзятости
• Применение: Анализ справедливости и валидация обнаружения предвзятости

Набор данных 3: Синтетические данные
• Генерация: Настраиваемый размер и сложность
• Задача: Контролируемая симуляция дрифта
• Признаки: Настраиваемые с демографическими атрибутами
• Применение: Валидация методов обнаружения дрифта


СЛАЙД 11: Описание данных - Набор данных Adult Income
--------------------------------------------------------------------------------
Детали набора данных Adult Income:

Источник: UCI Machine Learning Repository (OpenML)
Загрузка: Автоматическая через OpenML API
Хранилище: Кэшируется локально в data/raw/adult.pkl

Структура:
• Обучающая выборка: ~39,000 образцов (80%)
• Тестовая выборка: ~9,700 образцов (20%)
• Признаки: 14 атрибутов
  - Числовые: возраст, fnlwgt, education-num, hours-per-week и т.д.
  - Категориальные: workclass, education, occupation, race, sex и т.д.

Целевая переменная:
• Бинарная классификация: доход >50K (1) или ≤50K (0)
• Распределение классов: ~76% ≤50K, ~24% >50K (несбалансированное)

Доступные демографические атрибуты:
• Пол: Мужской, Женский
• Раса: Белые, Черные, Азиаты и т.д.
• Позволяет проводить анализ справедливости для разных групп

Детали набора данных COMPAS:

Ключевые атрибуты:
• Демография: Раса, пол, возрастные категории

Значение для исследования:

СЛАЙД 13: Описание данных - Синтетические данные
--------------------------------------------------------------------------------
Генерация синтетических данных:

Цель: Контролируемые эксперименты для валидации обнаружения дрифта

Метод генерации:
• sklearn.datasets.make_classification
• Настраиваемые параметры:
  - Количество образцов (по умолчанию: 10,000)
  - Количество признаков (по умолчанию: 10)
  - Информативные vs избыточные признаки
  - Контроль несбалансированности классов

Демографические признаки:
• Возраст: Случайное распределение (18-80)
• Пол: Бинарный (Мужской, Женский)
• Раса: Несколько категорий
• Позволяет тестировать справедливость без проблем конфиденциальности

Применение:
