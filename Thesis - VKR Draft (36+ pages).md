Актуальность

Внедрение моделей машинного обучения в производственные
системы сопровождается риском постепенной или резкой деградации
качества прогнозов. Одной из наиболее распространённых причин
является дрейф данных — расхождение статистических свойств
входных признаков (и, в ряде случаев, целевой переменной) между
обучающей выборкой и потоком данных в эксплуатации. В финансовом
секторе, системах противодействия мошенничеству и сервисах
принятия решений на основе табличных данных последствия
необнаруженного дрейфа включают рост числа ложных срабатываний,
пропуск новых типов мошенничества и снижение доверия к
автоматизированным решениям.

Современная практика MLOps предлагает множество специализированных
инструментов: платформы мониторинга дрейфа (Evidently AI, NannyML),
реестры моделей (MLflow), оркестраторы рабочих процессов (Apache
Airflow, Prefect). Однако для учебных проектов, прототипов и небольших
команд интеграция разрозненных решений требует значительных
инженерных затрат и не обеспечивает единой прослеживаемости от
сигнала дрейфа до развёртывания обновлённой модели.

Актуальной является разработка программного комплекса, объединяющего
детектирование дрифта, пороговые политики реагирования,
автоматизированное переобучение, версионирование артефактов и
визуальный интерфейс наблюдения в едином воспроизводимом стеке.

1. ИССЛЕДОВАНИЕ ПРЕДМЕТНОЙ ОБЛАСТИ

1.1. Деградация моделей машинного обучения в эксплуатации

При обучении с учителем модель аппроксимирует отображение
f: X → Y на обучающей выборке (X_train, Y_train), предполагая, что
распределение в эксплуатации совпадает с обучающим:
P_train(X, Y) = P_prod(X, Y). На практике это предположение
нарушается, что приводит к снижению метрик качества.

Выделяют несколько типов сдвига распределений:
— ковариатный сдвиг (data drift): меняется P(X), при сохранении
  P(Y|X);
— концептуальный дрейф (concept drift): меняется P(Y|X);
— сдвиг априорных вероятностей классов: меняется P(Y).

Для операционного мониторинга наиболее применим ковариатный дрейф,
поскольку его можно оценивать по входным признакам без немедленной
разметки исходов в production. Своевременное обнаружение сдвига
входных распределений позволяет инициировать переобучение до
критического падения бизнес-метрик.

Механизмы деградации включают внезапный дрейф (смена политики,
источника данных), постепенный дрейф (изменение клиентской базы),
сезонные колебания и смещение отдельных подпространств признаков.
В задачах бинарной классификации (доход, мошенничество) даже
умеренный сдвиг по ключевым признакам может существенно изменить
калибровку вероятностей модели.

1.2. Подходы к мониторингу распределений данных

Подходы к мониторингу классифицируют по используемой методологии.

Статистические тесты сравнивают эмпирические распределения reference
(обучающий или базовый период) и current (текущее окно). К ним
относят PSI, тест Колмогорова–Смирнова, критерий χ².

Метрики расстояния (Wasserstein, MMD, KL-дивергенция) измеряют
«стоимость» преобразования одного распределения в другое; они
чувствительны к форме распределения, но требуют калибровки порогов.

Модельные методы используют классификатор reference vs current или
мониторинг метрик качества на размеченных данных; последнее
выявляет concept drift, но зависит от доступности меток.

В производственных системах с высокой частотой запросов
предпочтительны интерпретируемые и вычислительно лёгкие методы
(PSI, KS, χ²), применяемые покомпонентно по признакам с последующей
агрегацией сигналов на уровне политики.

1.4. Статистические методы выявления дрифта

Population Stability Index (PSI) для бинов i с долями P_i (reference)
и Q_i (current):

  PSI = Σ (Q_i - P_i) · ln(Q_i / P_i)

Интерпретация (отраслевые эвристики): PSI < 0,10 — стабильно;
0,10 ≤ PSI < 0,25 — умеренный дрейф; PSI ≥ 0,25 — сильный дрейф.

Тест Колмогорова–Смирнова сравнивает эмпирические функции
распределения; признак считается значимым при p-value < α (по
умолчанию α = 0,05).

Критерий χ² проверяет однородность частот категорий между выборками;
редкие категории объединяются в класс «__other__» для выполнения
условий применимости теста.

В разрабатываемом комплексе для численных признаков применяются PSI
(по замороженным квантильным границам baseline) и KS; для
категориальных — PSI и χ².

1.5. Практики автоматизации переобучения моделей

Триггеры переобучения: по расписанию, по падению метрик качества,
по сигналу дрейфа. Дрейф-триггер не требует немедленных меток в
production, но должен сопровождаться правилом безопасного развёртывания.

Типовой конвейер: сбор данных → объединение reference и current →
обучение → оценка на holdout → сравнение с production-моделью →
условное продвижение. Shadow/canary-развёртывание в промышленности
снижает риск выката худшей модели; в прототипе реализовано правило
«продвигать только при macro-F1 ≥ чемпиона».

2.4. Проектирование оркестрации и политик

Класс Orchestrator.run_pipeline выполняет линейный конвейер:
1) загрузка матрицы признаков и разбиение reference/current по сценарию;
2) fit_or_load_baseline;
3) run_drift_analysis;
4) DriftThresholdPolicy.evaluate;
5) цепочка Action (Log, WebhookAlert, RetrainPipeline);
6) insert_run в RunStore.

Политика по умолчанию (строгое превышение):
— n_features_high_psi > 0;
— n_numeric_ks_significant > 2;
— n_categorical_chi2_significant > 3.

ЗАКЛЮЧЕНИЕ

В работе разработан программный комплекс для детектирования дрифта
данных и адаптивного переобучения моделей машинного обучения. Комплекс
объединяет статистический мониторинг (PSI, Kolmogorov–Smirnov, χ²),
пороговые политики, оркестрацию запусков, автоматизированное
переобучение на scikit-learn, реестр версий моделей с стадиями
жизненного цикла, провенанс данных и веб-дашборд.

Экспериментальная проверка на наборах Adult Census Income и Kaggle
Credit Card Fraud показала корректное поведение в сценариях
random_holdout, age_shift и темпоральных сравнениях fraud_d1_vs_d2,
fraud_d2_vs_d3, fraud_d1_vs_d3. Система не заменяет промышленные
MLOps-стэки, но демонстрирует целостную воспроизводимую архитектуру,
пригодную для дальнейшего развития (интеграция с Airflow/MLflow,
калибровка порогов на исторических данных, канареечное развёртывание).

Направления дальнейшей работы: обучаемые политики с учётом
бизнес-издержек; дрейф в пространстве эмбеддингов для мультимодальных
моделей (в т.ч. детекторы deepfake); развёртывание в Kubernetes и
переход с SQLite на промышленную СУБД.
